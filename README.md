# НормаОТ: Протокол ЭБ v0.9.6

**Протоколы ЭБ** — десктопное приложение (Electron + SQLite) для автоматизации учёта проверки знаний требований электробезопасности и формирования протоколов проверки знаний в формате DOCX.

Работает офлайн на ПК под Windows. Данные хранятся в локальной зашифрованной базе SQLite, обмен между рабочими местами — через FTP.

**Разработчик:** Кривоносов Д.А. · [denis.krv@](mailto:denis.krv@yandex.ru) · [normaot.ru](https://normaot.ru)

## Назначение

Программа заменяет ручное заполнение протоколов проверки знаний в Word и Excel. Она ведёт базу работников, справочники комиссий и прав, журнал проверок и график на год, а также формирует готовый протокол одним нажатием на основе шаблона `templates/Протокол.docx`.

## Возможности

- **Панель управления** — быстрое создание протокола по фамилии, сводка проверок по месяцам, кликабельные плитки со статусами.
- **Новый протокол** — пошаговая форма из 5 блоков: реквизиты, комиссия, проверяемый, нормативные документы и результаты, права работ. Живой предпросмотр и экспорт в DOCX.
- **База работников** — хранение ФИО, места работы, должности, предыдущих оценок и групп по ЭБ; импорт из Excel; сортировка по столбцам; экспорт с правами.
- **Журнал проверок знаний** — история всех проверок с экспортом, столбец «Причина».
- **График проверки знаний на год** — планирование сроков следующих проверок, экспорт в Excel.
- **Справочники** — комиссии (председатели и члены), подразделения, права работ, объёмы знаний, категории (в т.ч. «Инспектирующий»).
- **Синхронизация** — обмен данными между рабочими местами через FTP с поддержкой SOCKS/HTTP-прокси, диагностика подключения.
- **Безопасность** — шифрование локальной базы данных SQLite (PBKDF2+salt), хранение паролей в системном хранилище ключей.

## Технологический стек

| Компонент | Технология |
| --- | --- |
| Оболочка приложения | Electron 33 |
| База данных | SQLite (`better-sqlite3-multiple-ciphers`, шифрование) |
| Импорт Excel | `exceljs` |
| Генерация DOCX | `docxtemplater` + `pizzip` |
| Синхронизация | FTP (`basic-ftp`) |
| Хранение секретов | `keytar` (системное хранилище ключей) |
| Сборка | `electron-builder` (portable) |

## Быстрый старт

### Требования

- **Node.js** 20+ и **npm**
- **Windows x64** для сборки

### Установка и запуск

```bash
# 1. Клонировать репозиторий
git clone https://github.com/Kerlad/eb-protocols.git
cd eb-protocols

# 2. Установить зависимости
npm install

# 3. Создать базу данных
npm run init-db

# 4. (Опционально) импортировать данные из Excel
npm run import-excel -- "./import.xlsx"

# 5. Запустить приложение
npm start
```

### Команды npm

| Команда | Назначение |
| --- | --- |
| `npm start` | Запуск приложения (Electron) |
| `npm run init-db` | Создание/инициализация БД |
| `npm run import-excel -- <файл>` | Импорт работников и справочников из Excel |
| `npm run test-docx` | Проверка генерации DOCX по шаблону |
| `npm run build:win` | Сборка portable EXE для Windows |

## Руководство пользователя

### 1. Первый запуск и импорт данных

При первом запуске база данных создаётся автоматически. Для начала работы:

1. **Заполните справочники** в разделе «Справочники»: подразделения, комиссии (председатели и члены), права работ, объёмы знаний. Это **обязательно** — без справочников невозможно создать протокол.
2. Подготовьте Excel-файл с данными работников (можно создать шаблон кнопкой **«Создать шаблон»** в разделе «Работники»).
3. Перейдите в раздел **«Работники»** и нажмите **«Импорт Excel»**.
4. Выберите файл в диалоге. Программа загрузит работников и права из первого листа с данными.
5. После импорта данные появятся в таблице работников и на панели управления.

### 2. Формат импорта Excel

Импортируется первый лист с данными (название листа не важно). ФИО генерируется автоматически из фамилии, имени и отчества.

| Столбец | Содержимое | Пример |
| --- | --- | --- |
| 1 | Фамилия | Иванов |
| 2 | Имя | Иван |
| 3 | Отчество | Иванович |
| 4 | Место работы | Участок электрооборудования |
| 5 | Должность | Электромонтер |
| 6 | Объём знаний | Для эл.мех |
| 7 | Категория персонала | Оперативно-ремонтный |
| 8 | Группа | IV |
| 9 | Дата проверки знаний | 2026-06-15 |
| 10 | Дата следующей проверки | 2027-06-15 |
| 11 | Периодичность (лет) | 1 |
| 12+ | Столбцы прав работ (1 или «да» = право активно) | 1 |

### 3. Создание протокола

1. Перейдите в раздел **«Новый протокол»** или начните поиск на панели управления.
2. Введите фамилию работника — программа найдёт его в базе и предложит выбрать.
3. При выборе работника автоматически заполняются: ФИО, место работы, должность, группа ЭБ, категория, объём знаний, предыдущая проверка.
4. Проверьте и при необходимости измените: номер протокола, дату, причину, комиссию.
5. Выберите комиссию из списка — председатель и члены заполнятся автоматически.
6. Установите оценки по разделам (ЭБ, ОТ, ПБ, другие) и общую оценку.
7. Отметьте права работ чекбоксами.
8. Нажмите **«Предпросмотр»** — справа отобразится макет протокола.
9. Нажмите **«Сохранить DOCX»** для сохранения файла или **«Записать в журнал»** для записи в БД и генерации DOCX.

### 4. Управление работниками

- **Импорт Excel:** загрузка работников из Excel-файла (первый лист с данными).
- **Экспорт данных:** выгрузка текущей базы работников в Excel-файл.
- **Создать шаблон:** генерация пустого Excel-файла с заголовками столбцов для ручного заполнения.
- **Добавление:** нажмите «+ Добавить работника», заполните обязательные поля (ФИО), отметьте права.
- **Редактирование:** нажмите «Ред.» в строке работника.
- **Удаление:** нажмите «Уд.» — запись помечается как удалённая (мягкое удаление).
- **Поиск:** введите фамилию в поле фильтра — таблица фильтруется мгновенно.

### 5. Журнал проверок знаний

- Все созданные протоколы отображаются в разделе «Журнал ПЗ».
- Для каждого протокола доступна ссылка на файл DOCX.
- Кнопка **«Экспорт журнала»** сохраняет журнал в Excel-файл.
- **Удаление протокола:** нажмите «Удалить» в строке записи и введите слово `УДАЛИТЬ` для подтверждения. Нумерация при этом не пересчитывается — в месте удаления остаётся пропуск.

### Нумерация протоколов

- Нумерация начинается заново с №1 с начала каждого календарного года (1 января).
- Номер присваивается автоматически как `MAX(номер) + 1` за текущий год.
- При удалении записи нумерация не пересчитывается — пропуск сохраняется.

### 6. График проверок на год

- Раздел «График» показывает расписание проверок по месяцам.
- Выберите год — программа покажет всех работников с датой следующей проверки в этом году.
- В заголовке каждого месяца отображается количество работников. Нажмите на число — откроется окно со списком работников и датами проверок.
- Кнопка **«Excel»** экспортирует график в отформатированный Excel-файл.

### 7. Справочники

В разделе «Справочники» управляются:

- **Подразделения** — код, название, скрытие кода в протоколе.
- **Комиссии** — состав (председатель + до 3 членов), автозаполнение при выборе.
- **Председатели / Члены комиссии** — ФИО и должность.
- **Права работ** — название и текст для протокола.
- **Объёмы знаний** — код, наименование, перечень нормативных документов.

Для каждого справочника доступны: добавление, редактирование (кнопка «Ред.»), удаление (кнопка «Удалить»).

### 8. Настройки

**Синхронизация (FTP):**
- Заполните параметры FTP-сервера: хост, порт, логин, пароль, удалённый путь.
- Нажмите «Проверить связь» для тестирования подключения.
- «Выгрузить в облако» — отправляет текущую БД на сервер.
- «Загрузить из облака» — скачивает БД с сервера (текущая сохраняется в бэкап).

**Безопасность:**
- Введите пароль для шифрования БД и нажмите «Включить пароль».
- Для отключения шифрования нажмите «Отключить пароль».

**LibreOffice (опционально):**
- Укажите путь к `soffice.exe` для конвертации DOCX в PDF вручную.

### 8.1. Работа за корпоративным прокси (PAC/WPAD)

Приложение автоматически определяет системный прокси через Electron (`session.defaultSession.resolveProxy()`). Если прокси не обнаружен, проверяются переменные окружения `FTP_PROXY` / `HTTP_PROXY`.

**Поддерживаемые типы прокси:**
- **SOCKS4/SOCKS5** — через библиотеку `socks`. Рекомендуется для корпоративных сетей.
- **HTTP CONNECT** — туннель через HTTP-прокси. Работает для управления FTP, но канал данных PASV может не проходить через HTTP-прокси.

**Нестандартный порт:**
- Укажите порт FTP-сервера в настройках (по умолчанию 21). Прокси определяется для указанного порта.

**Известные ограничения:**
- FTP passive mode через HTTP-прокси: канал данных (PASV) создаёт второе TCP-соединение напрямую на сервер, минуя прокси. Если фаервол блокирует прямые подключения — используйте FTPS или SFTP.
- Рекомендуемый транспорт для строгих фаерволов: FTPS (явный TLS) или SFTP.

**Диагностика:**
- Нажмите «Диагностика» в разделе настроек для пошаговой проверки: прокси, TCP, FTP, итог с подсказками.

### 9. Экспорт и файлы

- **DOCX:** генерируется из шаблона `templates/Протокол.docx`. Файлы сохраняются в папку `data/protocols/`.
- **Excel (график):** экспортируется из раздела «График».
- **Excel (журнал):** экспортируется из раздела «Журнал ПЗ».
- **Excel (работники):** экспортируется из раздела «Работники».

### 10. Резервные копии

- Автоматический бэкап создаётся перед каждым скачиванием БД с FTP.
- Ручной бэкап: через настройки или при импорте данных.
- Восстановление из бэкапа доступно через настройки.
- Бэкапы хранятся в папке `data/backups/`.

```

  package.json
  app/                  # Electron: main, preload, renderer (UI)
    main.js
    preload.js
    renderer/

  backend/
    paths.js
    db/                 # схема, подключение, репозитории
    import/             # импорт Excel
    protocol/           # генерация DOCX, нумерация, плейсхолдеры
    sync/               # FTP, бэкапы, восстановление, конфликты

  templates/
    Протокол.docx       # шаблон протокола
  data/                 # БД, бэкапы, готовые протоколы (создаётся автоматически)
  docs/
```

## Шаблон протокола и плейсхолдеры

Файл `templates/Протокол.docx` содержит плейсхолдеры в фигурных скобках, которые подставляются генератором:

- **Реквизиты:** `{Номер}`, `{Дата}`, `{Причина}`, `{След_Дата}`
- **Комиссия:** `{Комиссия}`, `{Должность_ПК}`, `{ПК}`, `{Должность_ЧК_1}`, `{ЧК_1}`, `{Должность_ЧК_2}`, `{ЧК_2}`, `{Должность_ЧК_3}`, `{ЧК_3}`, `{инструкции}`
- **Проверяемый:** `{ФИО}`, `{Место_Работы}`, `{Должность}`, `{Дата_пред_проверки}`, `{пред_оценка}`, `{Объем_знаний}`
- **Результаты:** `{оценка_ЭБ}`, `{оценка_ОТ}`, `{оценка_ПБ}`, `{другие_ИОТ}`
- **Заключение:** `{оценка}`, `{Группа_ЭБ}`, `{Прод_Дублир}`, `{категория}`, `{права}`, `{категория_ЭУ}`

## База данных и безопасность

- Данные хранятся в локальном файле SQLite в каталоге данных приложения (EB_DATA_DIR / userData).
- БД может быть зашифрована паролем; ключ передаётся через опции конструктора `Database(path, {key})`.
- Пароли БД и FTP хранятся в системном хранилище ключей (keytar).
