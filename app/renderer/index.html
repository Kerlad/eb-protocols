О программе

Информация о программе и разработчике.

Назначение

Протоколы ЭБ — десктопное приложение для автоматизации учёта проверки знаний требований электробезопасности и формирования протоколов проверки знаний.

Программа заменяет ручное заполнение протоколов в Word и Excel. Она ведёт базу работников, справочники комиссий и прав, журнал проверок и график на год, а также формирует готовый протокол проверки знаний одним нажатием на основе шаблона.

Основные возможности

  • Панель управления — быстрое создание протокола по фамилии, сводка проверок по месяцам.
  • Новый протокол — пошаговая форма с живым предпросмотром и экспортом в DOCX.
  • База работников — хранение ФИО, мест работы, должностей, оценок и групп по ЭБ; импорт из Excel.
  • Журнал проверок знаний — история всех проверок с экспортом.
  • График проверки знаний на год — планирование сроков следующих проверок.
  • Справочники — комиссии, подразделения, права работ, объёмы знаний.
  • Синхронизация данных между рабочими местами через FTP.
  • Шифрование локальной базы данных SQLite.

Справка

Пошаговое руководство по работе в программе.

1. Импорт 2. Формат Excel 3. Протокол 4. Работники 5. Журнал 6. График 7. Справочники 8. Настройки 9. Экспорт 10. Бэкапы

1. Первый запуск и импорт данных

При первом запуске база данных создаётся автоматически. Для начала работы:

  1. Заполните справочники в разделе «Справочники»: подразделения, комиссии (председатели и члены), права работ, объёмы знаний. Это обязательно — без справочников невозможно создать протокол.
  2. Подготовьте Excel-файл с данными работников (можно создать шаблон кнопкой «Создать шаблон» в разделе «Работники»).
  3. Перейдите в раздел «Работники» и нажмите «Импорт Excel».
  4. Выберите файл в диалоге. Программа загрузит работников и права из первого листа с данными.
  5. После импорта данные появятся в таблице работников и на панели управления.

ФИО генерируется автоматически из фамилии, имени и отчества. Столбцы 12+ определяются как права работ.

3. Создание протокола

  1. Перейдите в раздел «Новый протокол» или начните поиск на панели управления.
  2. Введите фамилию работника — программа найдёт его в базе и предложит выбрать.
  3. При выборе работника автоматически заполняются: ФИО, место работы, должность, группа ЭБ, категория, объём знаний,.previous проверка.
  4. Проверьте и при необходимости измените: номер протокола, дату, причину, комиссию.
  5. Выберите комиссию из списка — председатель и члены заполнятся автоматически.
  6. Установите оценки по разделам (ЭБ, ОТ, ПБ, другие) и общую оценку.
  7. Отметьте права работ чекбоксами.
  8. Нажмите «Предпросмотр» — справа отобразится макет протокола.
  9. Нажмите «Сохранить DOCX» для сохранения файла или «Записать в журнал» для записи в БД и генерации DOCX.

4. Управление работниками

  • Импорт Excel: загрузка работников из Excel-файла (первый лист с данными).
  • Экспорт данных: выгрузка текущей базы работников в Excel-файл.
  • Создать шаблон: генерация пустого Excel-файла с заголовками столбцов для ручного заполнения.
  • Добавление: нажмите «+ Добавить работника», заполните обязательные поля (ФИО), отметьте права.
  • Редактирование: нажмите «Ред.» в строке работника.
  • Удаление: нажмите «Уд.» — запись помечается как удалённая (мягкое удаление).
  • Поиск: введите фамилию в поле фильтра — таблица фильтруется мгновенно.

5. Журнал проверок знаний

  • Все созданные протоколы отображаются в разделе «Журнал ПЗ».
  • Для каждого протокола доступна ссылка на файл DOCX.
  • Кнопка «Экспорт журнала» сохраняет журнал в Excel-файл.
  • Удаление: нажмите «Удалить» в строке протокола и введите слово УДАЛИТЬ для подтверждения. Нумерация не пересчитывается — пропуск сохраняется.

5.1. Нумерация протоколов

  • Нумерация начинается заново с №1 с начала каждого календарного года (1 января).
  • Номер присваивается автоматически как MAX(номер) + 1 за текущий год.
  • При удалении записи нумерация не пересчитывается — в месте удаления остаётся пропуск.

6. График проверок на год

  • Раздел «График» показывает расписание проверок по месяцам.
  • Выберите год — программа покажет всех работников с датой следующей проверки в этом году.
  • В заголовке месяца нажмите на число — откроется окно со списком работников и датами.
  • Кнопка «Excel» экспортирует график в отформатированный Excel-файл.

7. Справочники

В разделе «Справочники» управляются:

  • Подразделения — код, название, скрытие кода в протоколе.
  • Комиссии — состав (председатель + до 3 членов), автозаполнение при выборе.
  • Председатели / Члены комиссии — ФИО и должность.
  • Права работ — название и текст для протокола.
  • Объёмы знаний — код, наименование, перечень нормативных документов.

Для каждого справочника доступны: добавление, редактирование (кнопка «Ред.»), удаление (кнопка «Удалить»).

8. Настройки

Синхронизация (FTP):

  • Заполните параметры FTP-сервера: хост, порт, логин, пароль, удалённый путь.
  • Нажмите «Проверить связь» для тестирования подключения.
  • «Выгрузить в облако» — отправляет текущую БД на сервер.
  • «Загрузить из облака» — скачивает БД с сервера (текущая сохраняется в бэкап).

Безопасность:

  • Введите пароль для шифрования БД и нажмите «Включить пароль».
  • Для отключения шифрования нажмите «Отключить пароль».

LibreOffice:

  • Укажите путь к soffice.exe для автоматической генерации PDF (по умолчанию: C:\Program Files\LibreOffice\program\soffice.exe).

9. Экспорт и файлы

  • DOCX: генерируется из шаблона templates/Протокол.docx. Файлы сохраняются в папку data/protocols/.
  • Excel (график): экспортируется из раздела «График».
  • Excel (журнал): экспортируется из раздела «Журнал ПЗ».
  • Excel (работники): экспортируется из раздела «Работники».

10. Резервные копии

  • Автоматический бэкап создаётся перед каждым скачиванием БД с FTP.
  • Ручной бэкап: через настройки или при импорте данных.
  • Восстановление из бэкапа доступно через настройки.
  • Бэкапы хранятся в папке data/backups/.
