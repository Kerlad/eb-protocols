# История изменений

## v0.9.3

### Исправления импорта Excel

**Авто-определение сдвига данных:**
- Если Фамилия (столбец 1) отсутствует, но Имя (столбец 2) содержит значение — данные автоматически сдвигаются: Фамилия берётся из столбца 2, Имя — из столбца 3, Отчество — из столбца 4 и т.д.
- В отчёте импорта отображается предупреждение о сдвиге данных.

**Обработка просроченных дат:**
- Если расчётная дата следующей проверки оказывается в прошлом (меньше текущей даты), она автоматически пересчитывается как **текущая дата + 14 календарных дней**.
- В статусе работника отображается «Просрочено*» (красный цвет со звёздочкой).

**Модалка ошибок импорта:**
- Ошибки импорта отображаются в модальном окне (наследует тему приложения), а не в toast-уведомлении.
- Максимум 400px высота с прокруткой при большом количестве ошибок.

## v0.9.2

### Автоматические даты при импорте и ручном вводе

**Правило 1: Дефолтная дата последней проверки**
- Если при импорте у работника отсутствует дата последней проверки, она устанавливается на `01.01.2000`.
- Если дата последней проверки равна `01.01.2000`, дата следующей проверки рассчитывается как **текущая дата + 14 календарных дней**.

**Правило 2: Автоматический расчёт даты следующей проверки**
- Если при импорте (или ручном вводе) отсутствует дата следующей проверки, она рассчитывается автоматически:
  - **Административно-технический персонал:** дата проверки + 3 года
  - **Все остальные категории:** дата проверки + 1 год

**Затронутые файлы:**
- `backend/import/excelImporter.js` — логика нормализации дат при импорте
- `app/renderer/app.js` — аналогичная логика при ручном добавлении работника

- **[C1] Сдвиг колонок при импорте Excel** — исправлен маппинг в `dataSheetParser.js`: col4=Место работы (workplace_name + workplace_code), col5=Должность, col6=Объём знаний, col7=Категория, col8=Группа, col9=Дата проверки, col10=Дата следующей, col11=Периодичность, col12+=права. ### Улучшения безопасности
- **[H1] PBKDF2 вместо SHA-256** — `dbPassword.js`: ключ шифрования генерируется через PBKDF2 (100k итераций, случайная соль 16 байт). Соль хранится в keytar. Обратная совместимость со старыми БД.
- **[H2] TLS-сертификат по умолчанию проверяется** — `ftpClient.js`: `rejectUnauthorized: true` по умолчанию. Добавлена опция `allowSelfSigned` для самоподписанных сертификатов.
- **[H3] Сообщение о недоступности keytar** — `keyStorage.js`: добавлена `getAvailabilityMessage()` для вывода причины недоступности хранилища ключей.
- **[H4] EPSV→PASV fallback** — `ftpClient.js`: при ошибке EPSV автоматический повтор в PASV-режиме.

### Качество кода
- **[M1] Единый источник путей** — `connection.js` импортирует `paths.js` вместо дублирования логики.
- **[M2] Мёртвый код отчёта** — удалены неиспользуемые поля отчёта импорта (departments, chairmen, members, knowledgeScopes).
- **[M3] Singleton БД** — добавлена `getDb()` для долгоживущего соединения, `closeDb()` для закрытия.
- **[M4] Автотесты** — 10 тестов (protocolNumbering, placeholderMap, conflictDetector, rightsDetector, dataSheetParser). Запуск: `npm test`.
- **[M5] Версия из package.json** — `initDb.js` берёт версию из `package.json` вместо хардкода.
- **[M6] Удалён nsis-конфиг** — не использовался при target=portable.
- **[M7] Утилиты дат** — `backend/utils/dateUtils.js`: `formatDateLocal`, `addDaysLocal`, `addYearsLocal`, `todayLocal`. Работа в локальной таймзоне.
- **[M8] Дубли доков** — `docs/README.md` заменён на редирект в корневой README.

## v0.9.0

### Функциональность и новые возможности
- **Удаление протоколов** — запись из журнала удаляется по подтверждению (ввести «УДАЛИТЬ»). Нумерация не пересчитывается, пропуск сохраняется.
- **Sticky-шапки** — заголовки разделов и кнопки действий фиксируются при прокрутке: Новый протокол, Работники, Журнал, Справочники, Настройки, О программе, Справка.

- **Валидация формы протокола** — обязательные поля подсвечиваются красным при попытке сохранения.
- **Уведомление после сохранения** — вопрос «Открыть файл?» с кнопками Да/Нет.
- **Предпросмотр в отдельном окне** — кнопка «Предпросмотр» открывает DOCX в системном приложении.
- **Экспорт и шаблон работников** — кнопки «Экспорт данных» и «Создать шаблон» в разделе «Работники».
- **Импорт без ФИО** — столбец ФИО игнорируется, генерируется автоматически из ФИО.
- **Динамические годы в графике** — год текущий + следующий, при новом годе добавляется год +2.
- **Кликабельные месяцы в графике** — число работников открывает модалку со списком.
- **Тёмные диалоги** — prompt/confirm заменены на кастомные модалки с наследованием темы.
- **Справка с навигацией** — sticky-nav слева от текста, 10 разделов с якорями.
- **Обязательное заполнение справочников** — информация в Справке о необходимости заполнения перед импортом.

## v0.2.0

### Функциональность и новые возможности
- **Автоматическая генерация PDF** — добавлена CLI-интеграция с LibreOffice (`soffice.exe` в headless-режиме) для экспорта DOCX-протоколов в PDF при сохранении протокола или по нажатию в журнале.
- **Экран настроек приложения** — добавлен полноценный графический интерфейс настроек в боковое меню. Позволяет настраивать параметры FTP (хост, порт, учетные данные, пути), тестировать FTP-соединение, выполнять ручную выгрузку/загрузку базы данных, задавать путь к LibreOffice и настраивать шифрование/пароль локальной базы данных SQLite.
- **График проверки знаний на год** — реализован динамический календарь-график в разделе «График проверок» с группировкой сотрудников по месяцам и экспортом отформатированной таблицы в Excel (`exceljs`).
- **Справочник и управление комиссиями** — добавлена возможность выбора существующей комиссии из выпадающего списка при создании протокола с автоматическим заполнением должностей и ФИО председателя и членов комиссии. Добавлен интерфейс создания и удаления комиссий в справочниках.
- **Управление правами работников** — в модальное окно добавления/редактирования работника добавлены чекбоксы для редактирования его прав.
- **Регистронезависимый поиск на русском** — в SQLite зарегистрирована Unicode-совместимая функция `LOWER()` для кириллического поиска по ФИО.

### Исправления ошибок и оптимизация UI
- **Блокировка базы на Windows** — исправлен критический баг блокировки SQLite файла (`EBUSY`/`EPERM`) во время синхронизации: база данных закрывается перед FTP-копированием и открывается заново после.
- **Маппинг прав БД и UI** — исправлено расхождение названий прав (например, `'отв рук-ль работ;'` в БД и `'ответственный руководитель работ'` в UI), чекбоксы теперь корректно отображают и сохраняют права сотрудников.
- **Синхронизация удалений** — во все IPC-обработчики удаления справочников и сотрудников добавлена функция `touchDbModified(db)` для обязательного обновления ревизии БД перед синхронизацией.
- **Утечки соединений БД** — транзакция импорта Excel обернута в блок `try-finally` для гарантированного закрытия дескрипторов при ошибках.
- **Адаптивность и исправления CSS** — модальные окна получили вертикальную прокрутку и ограничение по высоте для ноутбуков с низким разрешением экрана. Добавлен адаптивный сайдбар и поддержка переноса текста кнопок меню на малых экранах. Предпросмотр А4 ограничен по максимальной ширине для естественного вида документа.

## v0.1.1

### Исправления ошибок

- **connection.js** — ключ шифрования БД теперь передаётся через опции конструктора `new Database(path, {key})` вместо нерабочего `PRAGMA key` после открытия. Устранена SQL-инъекция через интерполяцию ключа.
- **dataSheetParser.js** — `cellText()` корректно обрабатывает ячейки с формулами Excel (`{formula, result}`), которые ранее читались как `[object Object]`. Все 171 сотрудник теперь имеют корректные `full_name`.
- **syncService.js** — `downloadDb` после скачивания БД с FTP обновляет sync_state через свежее подключение, а не через устаревший in-memory handle.
- **excelImporter.js** — добавлен `db.close()` после завершения транзакции импорта (утечка файловых дескрипторов).
- **journalRepository.js** — удалён неиспользуемый импорт `touchDbModified`.
- **app.js** — добавлена защита от null при расчёте инициалов (`w.fio || "?"`).

### Исправление отображения работников

- Добавлен эндпоинт `employees:listAll` (repository, IPC handler, preload bridge).
- Таблица работников загружается через `listAll` вместо хака с поиском по 13 буквам (LIMIT 20 на каждый).

### Сборка

- `electron` перенесён из `dependencies` в `devDependencies` (требование electron-builder).
- Добавлен `signAndEditExecutable: false` для обхода ошибки symlink при сборке без Developer Mode.
- Добавлен `npmRebuild: false` — electron-builder's `@electron/rebuild` скачивал prebuilt для системного Node.js вместо Electron. Prebuilt для Electron подставляется вручную перед сборкой.
